# DeepOrra v0.1.0

**DeepOrra** is a local-first repository intelligence tool for AI coding agents. It indexes a Python repository on your machine and exposes its structure, symbols, routes, and code relationships through search, a dashboard, and read-only MCP planning tools so agents can check what already exists before writing new code.

This is the first public release of DeepOrra.

---

## Key Capabilities

- **Local-first indexing** — repository index data is stored in `.deeporra/` inside each indexed repository. No code leaves your machine.
- **Python AST analysis** — extracts functions, classes, methods, imports, and HTTP route definitions (Flask/FastAPI patterns).
- **Semantic embeddings** — local Sentence Transformers (`all-MiniLM-L6-v2`, 384 dimensions) run on CPU.
- **Three search modes** — lexical (FTS5 full-text), semantic (Chroma vector), and hybrid (combined) ranking.
- **Code graph** — relationships between files, symbols, imports, routes, and tests stored in SQLite `code_nodes`/`code_edges` tables.
- **Read-only MCP stdio server** — 8 planning tools for AI coding agents.
- **Streamlit dashboard** — localhost-only human-inspection UI for indexed repositories.
- **CLI** — five functional commands (`index`, `status`, `doctor`, `mcp`, `dashboard`) and one deferred stub (`setup`). Entry point: `deeporra` or `python -m deeporra`.

## Installation

Install from GitHub source:

```bash
git clone https://github.com/ashishkumar62649/deeporra.git
cd deeporra
python -m venv .venv
.venv\Scripts\activate
python -m pip install .
```

On Unix, activate the virtual environment with `source .venv/bin/activate`.

### Requirements

- Python 3.10+ with SQLite FTS5 support
- Core: typer, rich, chromadb, sentence-transformers, pydantic, python-dotenv, typing-extensions, mcp, streamlit
- Optional: tree-sitter and tree-sitter-python (multi-language parsing, deferred)
- 4-8 GB RAM recommended for indexing medium-sized repositories

## Quick Start

```bash
# Check your environment
deeporra doctor /path/to/repo

# Index a repository (full rebuild)
deeporra index /path/to/repo

# View index status
deeporra status /path/to/repo
```

If the console script is not on your PATH, use `python -m deeporra` instead of the bare `deeporra` command.

### Starting the MCP Server

Use either the CLI command or the module entry point:

```bash
deeporra mcp --repo /path/to/repo
# or
python -m deeporra.mcp_server
```

Configure your coding agent to launch this command as an MCP stdio subprocess. The server provides 8 read-only planning tools.

### Starting the Dashboard

Use either the CLI command or the module entry point:

```bash
deeporra dashboard --port 8501
# or
python -m deeporra.dashboard
```

Opens a local Streamlit app at `http://localhost:8501` (default port).

## MCP Tools (8)

All tools are served through a FastMCP stdio server. Every tool accepts `repository_root` (path to an indexed repository) and returns JSON. All tools are read-only.

| Tool | Purpose |
|------|---------|
| `repository_summary` | Return summary statistics for an indexed repository |
| `search_code` | Search code chunks by text, semantic, or hybrid mode |
| `hybrid_search` | Search code using combined text + semantic ranking |
| `find_symbols` | Find functions, classes, and methods matching a name |
| `find_routes` | Find HTTP route definitions by method, path, or handler |
| `get_related_code` | Find related code nodes through graph edges (one hop) |
| `analyze_change_impact` | First-order impact analysis for a symbol |
| `find_existing_implementation` | Search for existing code matching a description |

## Architecture Summary

```
          +----------+
          |   CLI    |
          +----+-----+
               |
       +-------v--------+
       |  Index Service  |
       |  (full rebuild) |
       +--+--+--+--+--+-+
          |  |  |  |  |
    +-----+  |  |  |  +------+
    v        v  v  v         v
 Scanner  Parser  Chunker  Embedder  Graph Builder
    |        |      |        |          |
    +--------+------+--------+----------+
                      |
              +-------v----------+
              |  Atomic Generation|
              |  Persistence      |
              +--+----+----+----+-+
                 |    |    |    |
           +-----v+ +v---+ +v--+ +v-----+
           |SQLite| |FTS5| |Chr| |Graph |
           |Store | |Store| |oma| |Store |
           +------+ +----+ +---+ +------+
                 |
          +------v----------+
          |  QueryService    |
          +--+-----------+--+
             |           |
      +------v--+  +----v------+
      |Dashboard |  | MCP Server|
      |Streamlit |  | 8 tools   |
      +---------+  +-----------+
```

DeepOrra runs entirely on your laptop. The CLI manages indexing, status, and health checks. The dashboard and MCP server use QueryService to read the active SQLite, FTS5, Chroma, and graph index stored in `.deeporra/`.

## Known Limitations

- **Python-only parsing** — AST support limited to Python. Multi-language parsing (TypeScript, Go, etc.) is deferred.
- **First-order impact only** -- `analyze_change_impact` shows direct relationships, not transitive analysis.
- **No incremental indexing** -- each `deeporra index` run performs a full rebuild.
- **One-hop graph** -- `get_related_code` supports `depth=1` only.
- **No automatic source editing** -- MCP tools are read-only and planning-only.
- **No private repository authentication** -- deferred to a later release.
- **No hosted SaaS** -- local-first only; no cloud deployment is planned.
- **No CI/CD integration** -- deferred to a later release.
- **Windows-tested only** -- primary development and testing on Windows. macOS and Linux not yet validated.

## Upgrade and Compatibility Notice

DeepOrra is in early development (v0.1.0). The indexing pipeline, CLI commands, MCP server, and dashboard are functional, but the API and data format may change in breaking ways before 1.0.0. Test thoroughly before relying on persisted indexes across versions.

There are no previous releases to upgrade from.
